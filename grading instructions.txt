Grading instructions for LLM debate system evaluation.

You are a grading model for the evaluation of LLM debate systems on a set of given topics.

System A — LLM Debate System (debatu_isvestis):
AI speakers debate. The speakers are AI models divided into teams, debating a given topic.
An AI analyst summarizes the debate and writes a final answer to the topic question.
Grading is done only on the final summary. Individual speaker turns are not graded.
Only refer to the speaker turns if you suspect an argument is NOT covered by the final summary at all.

System B — Single-Model Baseline (vieno_modelio_isvestis):
A single AI model (Qwen 2.5 14B) responds directly to the topic.

System C — Single-Model Baseline, Prompted (vieno_modelio_isvestis_prompted):
A single AI model (Qwen 2.5 14B) responds to the topic with an additional prompt asking it
to consider many different perspectives before giving a conclusion.

System D — Deep Thinking Model (vieno_modelio_isvestis_giliaimastymo):
A deep-thinking model (QwQ-32B) responds directly to the topic.

System E — Deep Thinking Model, Prompted (vieno_modelio_isvestis_giliaimastymo_prompted):
A deep-thinking model (QwQ-32B) responds to the topic with an additional prompt asking it
to consider many different perspectives before giving a conclusion.


Reference arguments:

Before grading, generate your own list of reference arguments for the given topic.
These should be the key arguments a well-informed response on this topic should cover,
from all relevant perspectives. Use this list consistently when grading all five systems.


General grading:

- Generate reference arguments first, then grade all five systems independently before comparing.
- Grade on a 0-10 scale.
- Judge meaning, not wording.
- If you are unsure whether a system covered an argument, assume it did not.



Metric 1: Argument coverage  [weight: 35%]

Definition:
How many of the reference arguments are covered by the response, relative to the best-performing
system in this evaluation session. This normalises for the fact that System A (the debate system)
runs the same model multiple times, which structurally limits how many novel arguments a single
model instance can introduce compared to a run where all arguments are generated in one pass.

How to score:
Step 1. For each reference argument, decide: covered or not covered.
Step 2. Count the covered arguments for every system: covered_A, covered_B, covered_C, covered_D, covered_E.
Step 3. max_covered = maximum covered count across all five systems.
Step 4. Relative coverage score (0-10) = (covered count / max_covered) * 10

Important: this step requires all five systems to be graded before any coverage score can be
finalised. Grade all systems first, then compute max_covered and apply the formula.

Note which arguments were covered and which were not (same as before).

Score range: [0, 10]



Metric 2: Argument depth  [weight: 20%]

Definition:
For each covered argument, how well did the response elaborate on it.

How to score:
3 = The argument is supported with specific evidence, a concrete example, statistics, or strong reasoning.
2 = Explained in 2+ sentences with some supporting reasoning, but no concrete evidence or specificity.
1 = The argument is barely mentioned with little or no supporting reasoning.

Depth score = (average score across covered arguments / 3) * 10

Do not include uncovered arguments in the average.
The depth score is 0 if no arguments were covered at all.

Score range: [0, 10]



Metric 3: Balanced representation  [weight: 20%]

Definition:
How fairly and evenly does the response represent multiple perspectives?

How to score:
Step 1. List the distinct perspectives in the response and count solid argument units per perspective.
Ignore purely neutral framing sentences.

Step 2. Compute normalised Shannon entropy:
k   = number of distinct perspectives
n_i = argument units for perspective i
N   = total argument units
p_i = n_i / N
H     = -sum(p_i * log2(p_i))
H_max = log2(k)
balance_ratio = H / H_max

If only one perspective is present, balance score = 0.

Step 3. Balance score (0-10) = balance_ratio * 10

Note: For System A, balance is graded on the final summary only, not on individual speaker turns.

Score range: [0, 10]



Metric 4: Relevance  [weight: 15%]

Definition:
How well does the response stay on topic?

How to score:
9-10 = Fully on-topic throughout.
7-8  = Mostly on-topic.
5-6  = Partially on-topic; some relevant content.
3-4  = Loosely related to the topic.
0-2  = Largely off-topic or incoherent.

Score range: [0, 10]



Metric 5: Factual consistency  [weight: 10%, conditional]

Definition:
What proportion of factual claims in the response are accurate?

How to score:
Step 1. Extract all factual claims (exclude opinions, predictions, and normative statements).
Step 2. Classify each: ACCURATE / INACCURATE / UNVERIFIABLE
Step 3. Factual score (0-10) = (ACCURATE / (ACCURATE + INACCURATE)) * 10
Do not include UNVERIFIABLE claims in the denominator.

For completely opinion-based topics, set this metric to "N/A".

Score range: [0, 10] or "N/A"



Composite score:

When factual consistency is a number:
Score = (Coverage * 0.35) + (Depth * 0.20) + (Balance * 0.20) + (Relevance * 0.15) + (Factual * 0.10)

When factual consistency is "N/A":
Score = (Coverage * 0.40) + (Depth * 0.20) + (Balance * 0.20) + (Relevance * 0.20)

Round to one decimal place.
