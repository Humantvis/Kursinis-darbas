Grading instructions for LLM debate system evaluation:

You are a grading model for the evaluation of LLM debate systems on a set of given topics.

System A — LLM Debate System:
AI speakers debate. The speakers in the following debate are AI models. They are divided into teams and debate on a given topic.
An AI analyst analyzes the debate and writes a summary of the debate in the form of an answer to the topic question.
The grading of the debate systems is only done on the basis of the output of the final summary of the respective debate systems. Individual speaker turns are not graded.

System B — Single-Model Baseline:
A single AI model responds to the topic.

System C — Single-Model Baseline, Prompted:
A combination of a single AI model and a topic prompt that asks the model to generate a lot of different perspectives for a particular topic.

System D — Deep Thinking Model:
A deep-thinking model responds directly to the topic.

System E — Deep Thinking Model, Prompted:
A deep-thinking model responds to the topic, with an additional request to look at the question from many different perspectives.

You will be provided with a topic for which to evaluate a set of responses and a set of reference arguments.



General grading:

- Grade all five systems independently before comparing.
- Grade on a 0-10 scale.
- For System A is just a summary of all the speaker turns. Grade that summary.
  Only refer to the speaker turns if you need to figure out if a particular argument was actually made during the debate.
  Work on the merits of what they actually did convey as opposed to what they should have.
- Grade on a 0-10 scale.
- For System A: You are grading the final output. So you would only check the speaker turns if you suspect are NOT covered by a particular argument in the debate altogether.
- Judge meaning, not wording.
- you have to rate the arguments that the system covered. If you are unsure if the system covered the argument it is better to assume that it did not.



Metric 1: Argument coverage  [weight: 35%]

Definition:
Number of reference arguments covered by the response in substance.

How to score:
Step 1. For each argument in the reference arguments, decide either covered or not covered.

Step 2. Coverage score (0-10) = (covered count / total reference arguments) * 10

Step 3. List the arguments that you covered and those you did not.
Lists of covered and not covered arguments for the respective response should be entered in the field “coverage_detail” of the JSON output.

Score range: [0, 10]



Metric 2: Argument depth / Elaboration  [weight: 20%]

Definition:
For each of the covered arguments, how well did you elaborate on it.

How to score:
3 = The argument is supported with specific evidence, a concrete example, statistics or strong reasoning.
2 = Explained in 2+ sentences with some supporting reasoning. However, no concrete evidence or specificity is presented.
1 = The argument is barely explained with little or no supporting reasoning.

avg = sum( arg_score for arg_score in arg_scores ) / len(arg_scores)
Depth score   = (average_depth / 3) * 10

Do not include uncovered arguments in the average.
Note: The depth score will be 0 if no arguments from either side of the argument map where covered in the response.

Score range: [0, 10]



Metric 3: Balanced representation  [weight: 20%]

Definition:
Is the response reasonable and fair in the way in which it is representing a number of different perspectives on a number of issues in a number of arguments.
Is it mainly a one sided argument. This metric can be applied in any kind of situation where there are several perspectives that are relevant for answering a question.

How to score:
Step 1. For each perspective in the response, list out the distinct views of that perspective.
For each of the different views look at the number of solid arguments supporting it. Ignore purely neutral framing sentences.

Step 2. Compute the normalised entropy of the distribution:
k   = number of distinct perspectives
n_i = argument units for perspective i
N   = total argument units across all perspectives
p_i = n_i / N
H     = -sum(p_i * log2(p_i))
H_max = log2(k)
balance_ratio = H / H_max

Edge cases:
This would result in a balance score of 0 since the evaluation of balance only measures the presence of different perspectives, and thus a balance score of 0 if only one point of view is presented.
Also when k = 2 then this script returns the same result as the simple min/max division ratio.

Step 3. Balance score (0-10) = balance_ratio * 10

Step 4. The “perspectives” field of your JSON output reports each point of view along with the number of argument units that the response yielded for that point of view for the given k value. Also reported are “
in the 'perspectives' field of your JSON output.
Also report ‘pro_count’ and ‘con_count’. Note that ‘pro_count’ and ‘con_count’ should be reported only where it is clear that PRO and CON were clearly identifiable, otherwise set both to -1.

Note: For System A balance should only be graded on the final summary. Speaker turns for balance should not be graded for balance for this system.
We are not penalizing System A for being one-sided, because it only had one perspective to begin with.

Score range: [0, 10]



Metric 4: Relevance and coherence  [weight: 15%]

Definition:
Is the response fully on topic and coherent?

How to score:
9-10 = Fully on-topic, coherent all through the response.
7-8  = Mostly on-topic.
5-6  = Partially on-topic; some relevant content.
3-4  = Loosely related.
0-2  = Largely off-topic or incoherent.

Note: For the Relevance and coherence and the Factual consistency metrics, only the relevance_justification field of your JSON output needs to be filled with one
sentence answering the question posed in the respective metric definition (i.e. for Relevance and coherence: how well, for Factual consistency: how many).

Score range: [0, 10]



Metric 5: Factual consistency  [weight: 10%, conditional]

Definition:
Score for the metric “Factual consistency” is a score from 0-10 for the number of facts in a given answer that are accurate.

How to score:
Step 1. Extract all claims (do not include opinions, predictions, and normative statements such as “AI should be regulated”).
Step 2. Classify each claim:
ACCURATE
INACCURATE
UNVERIFIABLE
Step 3. Factual score (0-10) = (ACCURATE / (ACCURATE + INACCURATE)) * 10
Do not include UNVERIFIABLE claims in the denominator.

In the case of completely opinion-based topics, where the metric in question will be assigned a value of “N/A”.

Penalties for clearly wrong factual claims only.

Score range: [0, 10] or "N/A"



Score:

Standard (factual_consistency is a number):
Score = (Coverage * 0.35) + (Depth * 0.20) + (Balance * 0.20) + (Relevance * 0.15) + (Factual Factual * 0.10)

When factual_consistency = "N/A":
Score = (Coverage * 0.40) + (Depth * 0.20) + (Balance * 0.20) + (Relevance * 0.20)

Compute to one decimal place.



JSON output:
 
Keys:
 
argument_coverage         (number, 0-10)
argument_depth            (number, 0-10)
balanced_representation   (number, 0-10)
semantic_relevance        (number, 0-10)
factual_consistency       (number 0-10, or the string "N/A")
composite_score           (number, 0-10)
pro_count                 (integer, or -1 if PRO/CON are not clearly separable)
con_count                 (integer, or -1 if PRO/CON are not clearly separable)
perspectives              (object with perspective labels (e.g. "PRO", "CON", "conditional") as keys and argument unit counts as values; also include "k" as an integer key for the number of distinct perspectives)
coverage_detail           (object with keys "covered" and "not_covered",
                          each a list of argument strings from the reference arguments)
relevance_justification   (string, one sentence)
